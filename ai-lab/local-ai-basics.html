Local AI Basics

Plain-language definitions for reading the model comparison tables

These definitions are written for people who are new to local AI. They are meant to explain the terms used around benchmarks, model tables, and tool workflows without requiring a machine learning background.

Local AI

AI models running on your own computer instead of a company's remote server. The tradeoff is more privacy and control, but your hardware limits speed, memory, and model size.

Cloud AI

AI services running on provider hardware, usually through a website or API. These can be very powerful, but your prompts and files may leave your machine depending on the service and settings.

Model

The AI system that reads your input and generates an output. In local AI, a model is often a downloadable file or group of files loaded by tools like LM Studio or Ollama.

Parameters

The learned internal values of a model. Parameter count is often shown as 3B, 7B, 13B, and so on. Bigger models can be more capable, but they usually need more memory and run slower.

Tokens

The chunks of text a model reads and writes. A token can be a whole word, part of a word, punctuation, or spacing. Token counts matter because context windows and pricing are usually token-based.

Context Window

The amount of text the model can consider at once, including your prompt, conversation history, files, and its own reply. Larger context helps with long tasks but often uses more memory.

Quantization

A compression method that makes a model smaller and easier to run locally. Lower quantization can save memory, but may reduce quality or make the model less stable.

GGUF

A common file format for running quantized local models with llama.cpp-based tools. If a model table lists GGUF, it usually means the model is packaged for local desktop use.

VRAM

Memory on your graphics card. For local AI, VRAM is often the tightest limit because loading more of the model onto the GPU usually makes generation faster.

RAM

Your system memory. If a model does not fully fit in VRAM, some work may fall back to RAM and CPU, which can be much slower.

Inference

The act of using a trained model to generate an answer. When you ask a model a question and it replies, that reply is produced through inference.

Benchmark

A repeatable test used to compare models. Benchmarks are useful for spotting patterns, but they do not replace trying a model on your actual work.

Eval

Short for evaluation. An eval may be a formal benchmark, a custom test set, or a practical workflow check designed to measure a model's behavior.

Hallucination

When a model confidently produces information that is wrong, unsupported, or made up. This is why important answers should be checked against reliable sources.

Tool Calling

When a model can ask software tools to do specific work, such as searching files, calling an API, running a calculation, or using a function provided by an app.

Agent

An AI workflow that can take multiple steps, use tools, inspect results, and continue toward a goal. Coding assistants are a common example of agent-style systems.

Reasoning Model

A model designed or tuned to spend more effort on multi-step problems. These can be better at planning, math, coding, and analysis, but may be slower.

Multimodal

A model that can work with more than text, such as images, audio, video, or files. For example, a multimodal model may inspect a screenshot and explain what it sees.

Open Weights

A model whose trained weight files are available to download. Open weights are not always the same as open source, because the license may still restrict some uses.
