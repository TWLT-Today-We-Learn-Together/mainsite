Local AI, plainly.

A practical glossary for reading model tables and trying local tools, with no prior machine-learning knowledge assumed.

Browse the glossaryWhere to go next
Local AI

Models running on your own computer instead of a company’s remote server. You gain privacy and control, while your hardware limits speed, memory, and model size.

Cloud AI

AI services running on provider hardware through a website or API. These can be powerful, but prompts and files may leave your machine depending on the service and settings.

Model

The AI system that reads your input and generates an output. Locally, it is often a downloadable file loaded by tools such as LM Studio or llama.cpp.

Parameters

Learned internal values, often shown as 3B, 7B, or 13B. Bigger models can be more capable, but generally require more memory and run slower.

Model architecture

The underlying design of a model—how its parts are arranged and how it processes information. Two models of the same size may behave differently because their architectures and training differ.

Tokens

Chunks of text a model reads and writes: whole words, word fragments, punctuation, or spacing. Context limits and pricing are commonly token-based.

Context window

How much text the model can consider at once, including prompts, history, files, and its reply. Larger context often needs more memory.

Quantization

Compression that makes a model smaller and easier to run. Aggressive quantization saves memory but can reduce quality or stability.

GGUF

A common file format for quantized models used by llama.cpp-based tools and many local desktop workflows.

VRAM

Memory on the graphics card. It is often the tightest local-AI limit; loading more of the model onto the GPU usually improves speed.

RAM

System memory. Models that do not fit in VRAM can use RAM and CPU, but usually run much slower.

Inference

Using a trained model to generate an answer. Every response produced by a loaded model is inference.

Inference server

Software that loads a model and makes it available to other programs, usually through a local web address or API. It is the bridge between the model file and an app or coding tool.

Prompt template

The formatting a model expects around instructions, messages, and tool results. A mismatched template can make a capable model answer poorly or ignore tools.

Benchmark

A repeatable test for comparing models. Benchmarks reveal patterns but do not replace testing your own work.

Eval

Short for evaluation: a formal benchmark, custom test set, or workflow check designed to measure model behavior.

Hallucination

When a model confidently produces wrong, unsupported, or invented information. Important outputs should be checked against reliable sources.

Tool calling

When a model can ask software tools to search files, call an API, calculate something, or run a function.

Agent

An AI workflow that takes multiple steps, uses tools, inspects results, and continues toward a goal. Coding assistants often work this way.

Reasoning model

A model designed or tuned to spend more effort on multi-step problems. It may improve planning, math, or code, but can run slower.

Multimodal

A model that works with more than text, such as images, audio, video, or files.

Open weights

A model whose trained weight files can be downloaded. This is not always the same as open source because licenses may still restrict use.
